# 本機 PDF 工具箱產品需求

## 產品目標

建立類似線上 PDF 工具網站的簡化版工具箱，但 PDF 文件全部留在使用者自己的電腦處理。第一階段面向不熟悉 Python、PowerShell或命令列的一般 Windows 使用者，功能正確與操作簡單優先於複製任何既有網站的外觀。

產品名稱為「本機 PDF 工具箱」。

## 目標環境與使用方式

- 第一階段支援 Windows 10／11 x64。
- 使用者第一次取得單一完整安裝程式 `本機PDF工具箱-安裝程式.exe`。
- 安裝程式建立桌面與開始功能表捷徑，並提供解除安裝。
- 安裝完成後不自動啟動工具或開啟瀏覽器；使用者之後自行雙擊捷徑啟動。
- 安裝後雙擊捷徑即可使用，不顯示 PowerShell 或命令提示字元。
- 工具自動在背景啟動本機服務，並以預設瀏覽器開啟操作介面。
- 桌面啟動器維持目前操作：服務準備完成後自動開啟預設瀏覽器，控制視窗保留重新開啟介面、檢查更新與結束工具。
- 使用者電腦不需要安裝 Python、uv、Streamlit、pypdf 或 pypdfium2。
- 安裝及日常 PDF 處理不需要網路；更新檢查及使用者前往 GitHub Releases 下載新版除外。
- 必須提供容易找到的「重新開啟介面」與「結束工具」操作。
- 結束工具時必須停止本機服務與背景程序；既有瀏覽器分頁應顯示工具已關閉的狀態，但不強制關閉一般瀏覽器分頁。

## 隱私與安全

- PDF、檔名與文件內容不得傳送至外部服務。
- 本機服務只監聽 `127.0.0.1`，不得暴露到區域網路。
- PDF 優先在記憶體處理；若未來功能必須使用暫存檔，完成或異常結束後應清除。
- 更新檢查不得包含 PDF、檔名或文件內容，也不建立非必要的裝置識別。
- 離線、更新伺服器失效或使用者拒絕更新時，已安裝功能仍可正常使用。
- `0.1.0` 先以未簽章公開測試版發布，必須清楚告知 Windows 可能警告或封鎖；程式只提供 HTTPS 更新提示及 GitHub Releases 連結，不自動下載或執行新版安裝程式。
- 未來若改為自動下載或自動執行更新，必須先恢復雜湊驗證與 Windows Authenticode 程式碼簽章。

## 功能需求

### 已完成：合併 PDF

- 一次上傳至少兩份 PDF。
- 顯示檔名與頁數。
- 上移、下移、移除及清除檔案。
- 依畫面順序合併，保留各頁尺寸、方向與視覺內容。
- 指定輸出檔名並下載結果。
- 支援中文與重複檔名。
- 損壞、空白、非 PDF 或加密檔案會中止整批合併，且不輸出不完整結果。

### 已確認、待實作：合併 PDF 介面改進

- 上傳元件只負責加入檔案；加入後立即清空元件顯示，所有排序、移除、預覽與合併均以單一 `pdf_items` 清單為準。
- 每份 PDF 顯示第一頁縮圖、順序、檔名、頁數、上移、下移與移除操作。
- 優先提供滑鼠拖曳排序，允許新增合適的 Streamlit 元件依賴；上移與下移按鈕保留為鍵盤可用及元件失敗時的替代操作。
- 使用唯一 ID 支援中文與重複檔名，排序、移除、清除與合併結果必須同步。
- 使用 `pypdf` 驗證、讀取頁數與合併；使用 `pypdfium2` 從記憶體中的 PDF bytes 低解析度渲染第一頁縮圖。
- PDF 與縮圖不寫入磁碟暫存。預覽必須逐份處理：完成一份 PDF 的第一頁渲染並關閉 document、page、bitmap 後，才處理下一份。
- 初始保護值為最多 50 份 PDF、總上傳容量 500 MB；縮圖目標寬度約 180～240 像素。實測顯示一般電腦仍有明顯記憶體壓力時再調整，不做超出必要範圍的最佳化。
- 損壞、空白、非 PDF 或加密檔案不產生縮圖，並沿用整批合併拒絕規則。

### 下一個規劃功能：拆分 PDF

- 依頁碼範圍輸出。
- 指定頁面另存 PDF。
- 支援每頁各自輸出；多檔結果使用 ZIP 下載。
- 驗證頁碼、範圍及空輸出。

### 後續候選功能

- 旋轉、刪除及重新排列頁面。
- 擷取指定頁面。
- 浮水印。
- 圖片轉 PDF、PDF 轉圖片。
- PDF 壓縮。

候選功能不代表已承諾時程；每次只新增一項可完整測試與發布的功能。

## 非目標

- 第一階段不複製 iLovePDF 的 UI 或全部功能。
- 第一階段不提供帳號、雲端儲存或多人協作。
- 第一階段不支援 macOS、Linux、Windows x86 或 Windows ARM 原生版本。
- 合併功能不承諾保留書籤、附件、互動表單、數位簽章或文件層級中繼資料。
