# 本機 PDF 工具箱產品需求

## 產品目標

建立類似線上 PDF 工具網站的簡化版工具箱，但 PDF 文件全部留在使用者自己的電腦處理。第一階段面向不熟悉 Python、PowerShell或命令列的一般 Windows 使用者，功能正確與操作簡單優先於複製任何既有網站的外觀。

產品名稱為「本機 PDF 工具箱」。

## 目標環境與使用方式

- 第一階段支援 Windows 10／11 x64。
- 使用者第一次取得單一完整安裝程式 `LocalPDFToolbox-Setup-v版本.exe`。
- 安裝程式與 SHA-256 清單檔名必須包含版本號；建立新版本時不得覆蓋本機或 GitHub Releases 中的舊版本。相同版本的重複候選建置可以更新同一版本檔案。
- 安裝程式建立桌面與開始功能表捷徑，並提供解除安裝。
- 安裝完成後不自動啟動工具或開啟瀏覽器；使用者之後自行雙擊捷徑啟動。
- 安裝後雙擊捷徑即可使用，不顯示 PowerShell 或命令提示字元。
- 工具自動在背景啟動本機服務，並以預設瀏覽器開啟操作介面。
- 桌面啟動器維持目前操作：服務準備完成後自動開啟預設瀏覽器，控制視窗保留重新開啟介面、檢查更新與結束工具。
- 使用者電腦不需要安裝 Python、uv、Streamlit、pypdf 或 pypdfium2。
- 安裝及日常 PDF 處理不需要網路；更新檢查及使用者前往 GitHub Releases 下載新版除外。
- 必須提供容易找到的「重新開啟介面」與「結束工具」操作。
- 結束工具時必須停止本機服務與背景程序；既有瀏覽器分頁只需能清楚辨識服務已停止或無法繼續操作，不要求顯示指定文字，也不強制關閉一般瀏覽器分頁。

## 隱私與安全

- PDF、檔名與文件內容不得傳送至外部服務。
- 本機服務只監聽 `127.0.0.1`，不得暴露到區域網路。
- PDF 優先在記憶體處理；若未來功能必須使用暫存檔，完成或異常結束後應清除。
- 更新檢查不得包含 PDF、檔名或文件內容，也不建立非必要的裝置識別。
- 離線、更新伺服器失效或使用者拒絕更新時，已安裝功能仍可正常使用。
- `0.1.0` 先以未簽章公開測試版發布，必須清楚告知 Windows 可能警告或封鎖；程式只提供 HTTPS 更新提示及 GitHub Releases 連結，不自動下載或執行新版安裝程式。
- 未來若改為自動下載或自動執行更新，必須先恢復雜湊驗證與 Windows Authenticode 程式碼簽章。

## 功能需求

### 已完成：合併 PDF

- 一次上傳至少兩份 PDF。
- 顯示檔名與頁數。
- 直接拖曳卡片調整順序，並可移除單一檔案或清除全部。
- 依畫面順序合併，保留各頁尺寸、方向與視覺內容。
- 指定輸出檔名並下載結果。
- 支援中文與重複檔名。
- 損壞、空白、非 PDF 或真正需要輸入非空密碼的檔案會中止整批合併，且不輸出不完整結果。
- 採寬鬆權限政策：若加密 PDF 可用空密碼直接開啟，就允許處理，不因禁止複製、修改或擷取等權限旗標而拒絕。

### 已完成：合併 PDF 介面改進

- 上傳元件只負責加入檔案；加入後立即清空元件顯示，所有排序、移除、預覽與合併均以單一 `pdf_items` 清單為準。
- 每份 PDF 顯示第一頁縮圖、順序、檔名、頁數與移除操作，不顯示上移或下移按鈕。
- 以滑鼠直接拖曳卡片作為主要排序操作；卡片依可用寬度自動排列為多欄、由左至右再由上至下，並支援跨列拖曳。
- 卡片較多時只在卡片區內捲動，避免整個工具頁面因檔案數量而無限制向下延伸。
- 使用唯一 ID 支援中文與重複檔名，排序、移除、清除與合併結果必須同步。
- 使用 `pypdf` 驗證、讀取頁數與合併；使用 `pypdfium2` 從記憶體中的 PDF bytes 低解析度渲染第一頁縮圖。
- PDF 與縮圖不寫入磁碟暫存。預覽必須逐份處理：完成一份 PDF 的第一頁渲染並關閉 document、page、bitmap 後，才處理下一份。
- 初始保護值為最多 50 份 PDF、總上傳容量 500 MB；PDFium 渲染縮圖目標寬度為 220 像素，極端長頁面另限制高度為 400 像素。實測顯示一般電腦仍有明顯記憶體壓力時再調整，不做超出必要範圍的最佳化。
- 損壞、空白、非 PDF 或真正需要輸入非空密碼的檔案不產生縮圖；可用空密碼直接開啟的權限保護 PDF 依寬鬆政策正常產生縮圖。

### 已完成：更新提示

- 啟動器在服務準備完成後自動檢查更新；同一天自動檢查至多一次，使用者仍可隨時按「檢查更新」重新檢查。
- 發現較新版本時顯示版本與發布說明，並可開啟 GitHub Releases 下載頁面。
- 程式不自動下載、不自動執行，也不刪除使用者下載資料夾中的舊安裝程式。
- 使用者關閉舊版後執行完整新版安裝程式；固定的 Inno Setup `AppId` 與安裝位置讓新版覆蓋既有安裝，不需要先解除安裝。
- `updates/update.json` 的版本必須高於已安裝版本才會顯示提示；公開資訊目前為 `0.2.0`，因此 0.1.0 會提示更新，0.2.0 不會重複提示。
- 每個新版發布前，須以舊版與新版完整安裝程式自動驗證原地覆蓋升級：安裝位置不變、主程式版本與內容已更新、只保留一筆解除安裝紀錄、兩版安裝後均未自動啟動且 `--self-test` 通過，最後可完整解除安裝及清理。驗證腳本只依賴 Windows PowerShell，不依賴 Python 或 uv。

### 0.2.0 已完成：PDF 轉圖片

- 一次可加入一份或多份 PDF，將每一頁完整渲染為一張圖片；用途包含把掃描 PDF 還原為逐頁圖片。
- 支援 PNG 與 JPEG（副檔名固定為 `.jpg`），可選擇 150、200 或 300 DPI；JPEG 可調整圖片品質。
- 所有結果以單一 ZIP 下載。ZIP 可選擇：
  - 每份 PDF 建立子資料夾，例如 `scan1/scan1-001.png`、`scan2/scan2-001.png`。
  - 全部圖片放在 ZIP 根目錄，例如 `scan1-001.png`、`scan2-001.png`。
- PDF 基礎名稱會移除不分大小寫的 `.pdf`，保留中文及名稱中的空格，將 Windows 不允許的 `< > : " / \ | ? *` 與控制字元改為 `_`，並移除開頭及結尾的空格與句點；清理後空白時使用 `document`。
- 清理後名稱以不分大小寫方式避免衝突；重複名稱依加入順序使用 `-2`、`-3` 等後綴。
- 圖片名稱固定為 `<唯一基礎名稱>-<頁碼>.<副檔名>`。頁碼從 1 開始，至少三位數（`001`）；超過 999 頁時自動增加位數。
- 子資料夾模式中的圖片仍保留 PDF 基礎名稱，不縮短為只有頁碼。
- ZIP 檔名可編輯。單一 PDF 預設為 `<唯一基礎名稱>-images.zip`，多份 PDF 預設為 `pdf-images.zip`；清理不安全字元並自動補上 `.zip`。
- 轉換前先驗證整批 PDF；只要有損壞、空白、非 PDF 或真正需要輸入非空密碼的檔案，就中止整批且不產生部分 ZIP；可用空密碼直接開啟的權限保護 PDF 依寬鬆政策允許轉換。
- PDF、渲染圖片與 ZIP 均不寫入磁碟暫存。轉換時一次只渲染一頁，寫入記憶體中的 ZIP 後立即關閉頁面、bitmap 與圖片物件，再處理下一頁。
- 以 40,000,000 像素作為單頁渲染保護上限，避免異常頁面尺寸耗盡記憶體；超過時顯示對應檔名與頁碼。
- 介面沿用第一頁縮圖、檔名、頁數、拖曳順序、移除與清除全部；拖曳順序同時決定 ZIP 內容順序及重複名稱後綴。

### 尚未排定：拆分 PDF

- 依頁碼範圍輸出、指定頁面另存及逐頁輸出仍為候選功能，尚未承諾版本。

### 後續候選功能

- 旋轉、刪除及重新排列頁面。
- 擷取指定頁面。
- 浮水印。
- 圖片轉 PDF。
- PDF 壓縮。

候選功能不代表已承諾時程；每次只新增一項可完整測試與發布的功能。

## 非目標

- 第一階段不複製 iLovePDF 的 UI 或全部功能。
- 第一階段不提供帳號、雲端儲存或多人協作。
- 第一階段不支援 macOS、Linux、Windows x86 或 Windows ARM 原生版本。
- 合併功能不承諾保留書籤、附件、互動表單、數位簽章或文件層級中繼資料。
